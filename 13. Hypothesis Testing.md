# Hypothesis Testing — Complete Guide

---

## Hypothesis Testing (Simple Explanation)

**Hypothesis testing** is a statistical method used to **make decisions about a population using sample data**.

In simple words:
We collect some data and check **whether a claim is likely true or not**.

Example:
A company claims **their light bulb lasts 1000 hours**.
We test this claim using sample data.

---

# 1. Null Hypothesis (H₀)

The **Null Hypothesis** represents the **default assumption** or **no change situation**.

It assumes **nothing unusual is happening**.

**Notation:** H₀

Example:

A company claims average bulb life = **1000 hours**

```text
H₀: μ = 1000
```

Meaning:
The average life of bulbs **is 1000 hours**.

---

# 2. Alternative Hypothesis (H₁ or Hₐ)

The **Alternative Hypothesis** is the **opposite of the null hypothesis**.
It represents the **claim we want to test**.

Example:

```text
H₁: μ ≠ 1000
```

Meaning:
The average life of bulbs **is NOT 1000 hours**.

---

### Types of Alternative Hypothesis

| Type         | Example  | Meaning                  |
| ------------ | -------- | ------------------------ |
| Two-tailed   | μ ≠ 1000 | Could be higher or lower |
| Right-tailed | μ > 1000 | Testing if it is greater |
| Left-tailed  | μ < 1000 | Testing if it is smaller |

---

# 3. Level of Significance (α)

The **Level of Significance** is the **maximum probability of rejecting a true null hypothesis**.

Common values:

```text
α = 0.05  (5%)
α = 0.01  (1%)
```

Meaning:

If **α = 0.05**, we allow **5% chance of making a wrong decision**.

Simple interpretation:

If probability of error is **less than 5%**, we reject the null hypothesis.

---

# 4. p-value

The **p-value** tells us **how strong the evidence is against the null hypothesis**.

Definition:

**p-value = probability of getting the observed result assuming H₀ is true**

Decision rule:

```text
If p-value ≤ α  → Reject H₀
If p-value > α  → Fail to reject H₀
```

Example:

α = 0.05

Case 1:

```text
p-value = 0.02
```

Since **0.02 < 0.05**

→ Reject H₀

Case 2:

```text
p-value = 0.08
```

Since **0.08 > 0.05**

→ Do not reject H₀

---

# 5. Type I Error

A **Type I Error** occurs when we **reject a true null hypothesis**.

In simple words:

We say something **changed**, but actually **nothing changed**.

Example:

A patient is healthy but the test says **disease present**.

Probability of Type I error:

```text
P(Type I error) = α
```

---

# 6. Type II Error

A **Type II Error** occurs when we **fail to reject a false null hypothesis**.

In simple words:

We say **nothing changed**, but actually **something changed**.

Example:

A patient has disease but the test says **healthy**.

Probability:

```text
P(Type II error) = β
```

---

# Type I vs Type II Error

| Error         | Meaning         | Example                       |
| ------------- | --------------- | ----------------------------- |
| Type I Error  | Reject true H₀  | Healthy person diagnosed sick |
| Type II Error | Accept false H₀ | Sick person diagnosed healthy |

---

# Simple Real-Life Example

Suppose a **teacher claims average marks = 70**

### Step 1: Hypothesis

```text
H₀: μ = 70
H₁: μ ≠ 70
```

### Step 2: Significance level

```text
α = 0.05
```

### Step 3: Test result

```text
p-value = 0.03
```

### Step 4: Decision

```text
0.03 < 0.05
```

→ Reject H₀

Conclusion:

Average marks **are different from 70**.

# One-Tailed Test and Two-Tailed Test (Simple Explanation)

In **hypothesis testing**, the **tail** refers to the **extreme ends of a probability distribution curve**.

When we test a hypothesis, we check whether the value lies in the **extreme region (tail)** of the distribution.

There are **two types of tests**:

1. **One-Tailed Test**
2. **Two-Tailed Test**

---

# 1. One-Tailed Test

A **One-Tailed Test** checks for a difference in **only one direction**.

We test whether a value is **either greater than or less than a certain value**, but **not both**.

There is **only one rejection region (one tail)**.

### Types of One-Tailed Tests

| Type         | Hypothesis | Meaning                     |
| ------------ | ---------- | --------------------------- |
| Right-tailed | H₁: μ > μ₀ | Testing if value is greater |
| Left-tailed  | H₁: μ < μ₀ | Testing if value is smaller |

---

### Example (Right-Tailed Test)

A company claims a battery lasts **10 hours**.

We want to check if it lasts **more than 10 hours**.

Hypothesis:

```text
H₀: μ = 10
H₁: μ > 10
```

We only check the **right side of the distribution**.

If the value falls in the **right tail**, we reject H₀.

---

### Example (Left-Tailed Test)

A company claims the average delivery time is **30 minutes**.

We want to check if delivery is **faster**.

```text
H₀: μ = 30
H₁: μ < 30
```

We check the **left tail**.

---

# 2. Two-Tailed Test

A **Two-Tailed Test** checks for a difference in **both directions**.

We test whether the value is **either greater OR smaller** than a given value.

There are **two rejection regions** (both tails).

---

### Hypothesis

```text
H₀: μ = μ₀
H₁: μ ≠ μ₀
```

Meaning:

The value could be **higher OR lower** than expected.

---

### Example

A company claims average salary is **₹50,000**.

We want to check if the salary is **different from ₹50,000**.

```text
H₀: μ = 50000
H₁: μ ≠ 50000
```

Now we check **both sides** of the distribution.

If the value is **too high OR too low**, we reject H₀.

---

# One-Tailed vs Two-Tailed Test

| Feature                | One-Tailed Test           | Two-Tailed Test         |
| ---------------------- | ------------------------- | ----------------------- |
| Direction tested       | One direction             | Both directions         |
| Alternative hypothesis | μ > μ₀ or μ < μ₀          | μ ≠ μ₀                  |
| Rejection region       | One tail                  | Two tails               |
| Example                | Check if profit increased | Check if profit changed |

Suppose a teacher says:

Average marks = **70**

### One-tailed test

Check if marks **increased**

```text
H₀: μ = 70
H₁: μ > 70
```

### Two-tailed test

Check if marks **changed**

```text
H₀: μ = 70
H₁: μ ≠ 70
```

# One-Line Summary

**One-Tailed Test:** Tests change in **one direction** (greater OR smaller)

**Two-Tailed Test:** Tests change in **both directions** (different)
